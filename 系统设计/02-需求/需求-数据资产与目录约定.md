# 需求：数据资产与目录约定（DataAssets）

| 字段 | 值 |
|------|-----|
| 模块键 | DataAssets |
| 状态 | agreed（与仓库现状对齐，路径以迭代为准） |

## 1. 目标

为 **ScriptableObject / 预制体引用数据** 建立可检索的目录约定与命名习惯，使策划、程序与构建流水线对「数据放哪、谁创建、谁引用」有统一预期；支持数万玩家规模下的 **版本化内容扩展** 与 **测试数据隔离**。

## 2. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-DAT-001 | 对话与消息相关资产具备稳定子目录（如 `04_Data/对话dat/` 下频道配置、角色库、剧情演出） | P0 | 与 `ChatChannelRegistrySO` 等 IMPL 描述一致 |
| REQ-DAT-002 | 标签类 `EmployeeTagSO` 资产可分区存放（如按 facet 分子文件夹），`tagId` 全局唯一 | P0 | 运行时解析无歧义 |
| REQ-DAT-003 | 测试用资产以命名或子路径区分（如 `*_Test`），正式包体可剥离或替换 | P1 | 发行配置不引用测试库 |
| REQ-DAT-004 | 编辑器菜单创建路径与运行时加载路径一致或可文档化映射 | P1 | 见 `ShenRenShiBuAssetMenu` 内常量；若存在 `00_Data` 与 `04_Data` 并存，需在矩阵备注收敛策略 |

## 3. 非功能需求

- 所有资产提交时 **.meta 同行**，避免 GUID 丢失导致引用大面积断开。
