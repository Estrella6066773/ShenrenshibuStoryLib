# 需求：核心消息与启动门控（CoreMessaging）

| 字段 | 值 |
|------|-----|
| 模块键 | CoreMessaging |
| 状态 | agreed（与当前实现一致） |

## 1. 目标

为全游戏提供**类型安全、可替换实现**的全局事件总线，并在运行时启动阶段提供**统一就绪信号**，避免委托等重系统在基础设施未安装前订阅命令。

## 2. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-CORE-001 | 存在全局总线安装点，安装后 `RuntimeMessageHub.IsReady == true` | P0 | 任意业务系统在安装后可 `Publish/Subscribe` |
| REQ-CORE-002 | 广播「运行时初始化完成」事件，携带 `gateKey` 与来源标识 | P0 | 委托管理器可配置为仅在该 gate 通过后接入创建命令 |
| REQ-CORE-003 | 总线实现支持订阅 token 生命周期管理（避免泄漏） | P1 | 重复进入场景无异常订阅堆积（需结合具体场景验证） |

## 3. 非功能需求

- 初始化顺序：`EventBusBootstrap` 应早于依赖总线的 `DefaultExecutionOrder` 系统执行（见 IMPL）。

## 4. 依赖

- 无上游游戏模块依赖；被几乎所有子系统依赖。
