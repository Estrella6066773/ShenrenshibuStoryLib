# 需求：员工模型与标签 Buff（Employee）

| 字段 | 值 |
|------|-----|
| 模块键 | Employee |
| 状态 | agreed（与当前实现一致） |

## 1. 目标

表示员工**能力、状态、分配关系、标签**；提供控制器层变更入口；通过注册表通知视图与任务系统；标签可叠加 **Buff** 影响能力、日薪、个人效率、压力获得等通道。

## 2. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-EMP-001 | 员工状态包含：闲置、委托中、待处理、矛盾中、离职等（以枚举为准） | P0 | 状态机与 UI/委托读取一致 |
| REQ-EMP-002 | 分配/解除分配/待处理/开除等操作经 `EmployeeController`，并触发注册表变更通知 | P0 | 视图与委托侧数据同步 |
| REQ-EMP-003 | 标签 Buff 从 `EmployeeTagSO.buffModifiers` 聚合为快照并应用到 `EmployeeData` | P0 | 增删标签后能力与薪资按规则变化 |
| REQ-EMP-004 | 开除计算赔偿金等业务规则（以代码公式为准） | P1 | 与策划表一致 |

## 3. 非功能需求

- 标签解析通过 `ITagBuffResolver`（可替换数据源）；聚合逻辑纯静态工具类，便于测试。
