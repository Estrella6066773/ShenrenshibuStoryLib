# 需求：编辑器工具链（EditorTools）

| 字段 | 值 |
|------|-----|
| 模块键 | EditorTools |
| 状态 | agreed（与当前实现一致） |

## 1. 目标

在 Unity 编辑器内提供 **资源创建菜单**、**关键 SO 的自定义 Inspector**、**场景快速搭建**、**策划数据管线（如 CSV→JSON）**，降低配置错误率并与运行时模块契约一致。

## 2. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-EDT-001 | 「神人事部」菜单下可创建标签库、标签、简历模板、委托分支树、频道/角色/剧情/邮件等资产，路径符合工具内常量 | P0 | 创建后类型与默认字段可保存 |
| REQ-EDT-002 | 委托系统场景搭建菜单可生成/补齐委托世界根、模板子物体与核心 Behaviour 引用链（见工具说明） | P1 | 新场景 Play 不报缺引用（在已绑定预制前提下） |
| REQ-EDT-003 | `AssignmentTemplateSO`、`ChatStorySO`、`MailEventDatabaseSO` 等具备 Editor 扩展时，Inspector 操作不破坏序列化引用 | P1 | 保存后 diff 仅预期字段 |
| REQ-EDT-004 | CSV 转 JSON 等批处理工具在失败时给出可读日志，不写坏源文件 | P2 | 见 `MailEventCsvToJsonConverter` 行为 |

## 3. 非功能需求

- 编辑器脚本仅存在于 `Editor` 程序集，不污染 Player 构建（Unity 默认规则）。
