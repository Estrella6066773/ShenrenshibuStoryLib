# 需求：玩家经济与日历时钟（PlayerEconomy）

| 字段 | 值 |
|------|-----|
| 模块键 | PlayerEconomy |
| 状态 | agreed（与当前实现一致） |

## 1. 目标

统一管理**虚拟日期推进、日长度、资金、日结扣薪、任务完成/失败结算**，并向其他系统（如矛盾网关）暴露只读或受控接口。

## 2. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-PES-001 | 可配置「现实秒 ↔ 游戏日」比例（默认 60s=1 天，可由 SO 覆盖） | P0 | 修改配置后跨天行为符合 SO |
| REQ-PES-002 | 跨天时触发日结逻辑（扣薪等），并通知依赖方 | P0 | 资金与日计数变化可观测 |
| REQ-PES-003 | 任务终态结算幂等：同一任务同一终态不重复发钱 | P0 | 重复触发结算接口资金不叠加 |
| REQ-PES-004 | 维护玩家性格标签 ID 与可选 SO 引用；维护「异常」标签 ID 列表（如压力后果） | P1 | UI 可展示；读档后 ID 与 SO 不一致时有定义行为（清空不匹配引用） |

## 3. 非功能需求

- 日薪缓存：员工数据变更时置脏，避免每帧全量遍历（性能）。
