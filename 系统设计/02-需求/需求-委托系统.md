# 需求：委托系统（Assignment）

| 字段 | 值 |
|------|-----|
| 模块键 | Assignment |
| 状态 | agreed（与当前实现一致） |

## 1. 目标

支持从模板**创建委托实例**、运行时 **Tick** 推进阶段与任务、**分支树裁决**终态、阶段 **hook** 驱动叙事/UI；管理器侧支持**初始化门控**、命令幂等与错误码、任务内**标签交互效果引擎**（如傲慢/嫉妒等规则）。

## 2. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-ASG-001 | 未收到约定 `gateKey` 的初始化完成事件前，拒绝业务创建命令并返回可诊断错误 | P0 | 与测试 Bootstrap 门控行为一致 |
| REQ-ASG-002 | 创建实例后发布创建/状态变更/终态等消息，供 UI 与其余系统订阅 | P0 | 消息载荷含 assignmentId 等关键键 |
| REQ-ASG-003 | 单实例 `AssignmentController.Tick` 推进倒计时、职位进度、分支评估 | P0 | 完成/失败/过期等终态可达成 |
| REQ-ASG-004 | 职位进度综合基础速度、倍率、个人能力、任务内临时效果等（以代码为准） | P0 | 与 `EmployeeWorkStateResolver`、矛盾/压力桥接一致 |
| REQ-ASG-005 | 任务槽位上员工集合参与矛盾涉事名单等集成 | P1 | 与 Conflict 模块文档一致 |
| REQ-ASG-006 | 任务启动就绪校验（若启用）在创建路径上执行 | P1 | 不满足条件时拒绝创建并给出原因 |

## 3. 非功能需求

- `businessUid` 等键符合管理器内正则约束，避免存档/日志注入混乱。
