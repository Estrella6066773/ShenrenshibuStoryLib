# 需求：运行时测试辅助（RuntimeTests）

| 字段 | 值 |
|------|-----|
| 模块键 | RuntimeTests |
| 状态 | agreed（与当前实现一致） |

## 1. 目标

在 **Play Mode** 下可重复验证 **总线门控**、**委托创建命令**、**消息闭环**，不依赖正式 UI 流程即可观察日志或状态；默认关闭，避免污染正式场景。

## 2. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-TST-001 | `AssignmentTestBootstrap` 可选发布 `RuntimeInitializationCompletedEvent` 并发送 `CreateAssignmentCommandEvent` | P0 | 与 `AssignmentManager` 门控配置一致时可创建实例 |
| REQ-TST-002 | `AssignmentPlaymodeVerifier` 可订阅委托接受/拒绝/创建/终态等事件并输出日志 | P1 | `enableVerifier` 为 false 时零开销订阅 |
| REQ-TST-003 | `StartupInitAssignmentTestSender` 在 `AssignmentManager.IsReady` 等条件满足后发送首条创建命令，支持有限帧重试 | P1 | `enabledInPlayMode` 默认 false |

## 3. 非功能需求

- 测试组件应带明确 `[DefaultExecutionOrder]` 或文档说明，避免与正式 Bootstrap 顺序冲突。
