# 需求：矛盾系统（Conflict）

| 字段 | 值 |
|------|-----|
| 模块键 | Conflict |
| 状态 | agreed（与当前实现一致） |

## 1. 目标

在任务域内累积矛盾，达阈值后从模板池生成矛盾事件，支持**玩家协调结算**、**冷却/去重**、**日终超时**，并通过桥接将涉事员工置于 **`InConflictEvent`**，与委托产出规则对齐。

## 2. 功能需求

| ID | 描述 | 优先级 | 验收要点 |
|----|------|--------|----------|
| REQ-CNF-001 | `ConflictSystemService` 为编排核心：累积、触发、回落、结算幂等 | P0 | 重复结算键不重复扣款/发奖 |
| REQ-CNF-002 | 依赖经济网关、压力网关与事件总线接口（可替换实现） | P0 | 与 `PlayerEconomySystem`、`StressSystem` 集成无硬编码单例耦合（以构造注入为准） |
| REQ-CNF-003 | 存档映射：矛盾状态可序列化/恢复（见 Save 层） | P1 | 读档后进行中案件一致 |
| REQ-CNF-004 | 与邮件等叙事桥接（`ConflictMailBridge` 等）可配置 | P2 | 触发时可投递预期邮件 id |
