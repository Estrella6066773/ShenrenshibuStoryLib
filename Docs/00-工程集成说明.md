# Unity 工程集成说明

## 版本控制原则（必读）

| 范围 | 工具 | Git 是否提交 |
|------|------|----------------|
| **`Docs/`**（系统设计、程序设计、设定设计） | **Git**（仓库在 Unity **根目录**） | 是 |
| **`.github/`、`Scripts/`、`.cursor/rules/`** 等文档配套 | Git | 是 |
| **`Assets/`**（脚本、场景、Prefab、SO、美术） | **UVC**（Unity Version Control） | **否** |
| **`ProjectSettings/`** | UVC | **否** |

**一个根目录 Git 仓库**，职责**仅限文档与相关配置**；程序实现与 Unity 资源**不由 Git 管理**，与 [机魂](D:\Unity\机魂) 相同。

`.gitignore` 已排除 `Assets/`、`ProjectSettings/`；`ignore.conf` 已排除 `/.git`、`/.github`，避免 UVC 与 Git 冲突。

## 目录

```
神人事部/                 ← Git 根（只提交文档侧）
├── Docs/
│   ├── 系统设计/
│   ├── 程序设计/
│   └── 设定设计/
├── .github/workflows/
├── Scripts/Get-DocsDate.ps1
├── Assets/               ← UVC，勿 git add
└── ProjectSettings/      ← UVC
```

## 日常操作

```powershell
# 在 Unity 工程根目录
cd "d:\Unity\神人事部"

# 只改文档时
git add Docs/
git commit -m "docs: …"
git push

# 取「最后更新」日期
powershell -NoProfile -File Scripts\Get-DocsDate.ps1
```

**不要**对 `Assets/`、`ProjectSettings/` 执行 `git add`（已被忽略；若误加，用 `git rm -r --cached` 撤回）。

## 改文档后的习惯

1. 在 `Docs/` 对应三分库内修改并提交。  
2. 若改玩法/实现：同步 `系统设计/02-需求` 与 `程序设计/04-实现`，更新 `程序设计/05-交付/交付-追溯矩阵.md`。
