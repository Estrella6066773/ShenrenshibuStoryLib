# 导出与一致性检查

## 第一条　导出管线（占位）

游戏引擎、Wiki 或本地化工具链的字段映射与构建脚本，建议放在本目录下的子文件夹（例如 `导出管线/`，等工具链落地时再建），并与 [玩家投放/vx.x/manifest.yaml](../玩家投放/v0.1/manifest.yaml)（版本号随升版替换）对齐。

注册表与命名规则见 [治理/README.md](../治理/README.md)。

## 第二条　一致性检查清单（人工／半自动）

建议在每次升版前执行：

1. [治理/id-registry.yaml](../治理/id-registry.yaml) 里的 `id` 与 [设定真值/](../设定真值/) 各文件 YAML 头里的 `id` 一致；  
2. [玩家投放/vx.x/manifest.yaml](../玩家投放/v0.1/manifest.yaml)（示例为 v0.1）所列 `id` 均存在于注册表；  
3. [对外读本](../玩家投放/v0.1/对外读本/00-世界观导览与阅读分层.md) 所引用的制度陈述都要有 [设定真值](../设定真值/README.md) 对应条文支撑，避免未经核对的口径漂移；  
4. 已废弃条目若仍出现在 manifest 引用里，要报错或触发升版流程；  
5. **内部链接**：`python 正文/管线/检查/check-links.py` 须退出码为 0。已配置钩子时，每次 `git commit` 会先 **`--fix --stage-fixed`**（按文件名／`id` 自动改链并 `git add` 正文变更，不 push）；无法修复的仍报错。安装钩子：[管线/检查/install-git-hook.ps1](../管线/检查/install-git-hook.ps1) 或 `git config core.hooksPath 正文/管线/检查`。书写约定见 [治理/03-内部链接书写规则.md](../治理/03-内部链接书写规则.md)。

## 第三条　与叙事生产稿的接口

从 [叙事生产稿/](../叙事生产稿/README.md) 导出到游戏内时，应附带 `canon_refs` 列表（字段名可保留英文以利工具链），便于回归测试与考据者溯源（若产品策略允许公开溯源链）。
