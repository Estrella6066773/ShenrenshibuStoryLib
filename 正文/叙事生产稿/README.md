# 叙事生产稿（非真值层）

本目录存放**任务简报、脚本、关卡叙事、活动文案**等随版本迭代更新的生产稿。此类文档：

- 允许标注为草案、分支线或已废弃；  
- 与 `设定真值/` 冲突时，**以 `设定真值/` 为准**，除非已获书面特批并登记例外编号；  
- 不得被考据者误认为「设定真值」；对外引用要经过 `玩家投放/` 冻结层转写。

## 子目录约定

| 子目录 | 用途 |
|--------|------|
| `01-简报/` | 任务与活动的策划简报。 |
| `02-脚本/` | 台本、分支对话、演出说明。 |
| `03-节拍/` | 可选：**叙事节拍**（story beats）：剧情先后次序与转折点的列表；以及大纲、结构草图（不是音乐「拍子」，而是剧情骨架）。 |
| `99-归档/` | 整批过期内容的归档迁移，保留决策痕迹（序号置末，避免与在线生产稿混淆）。 |

已归档批次见 [99-归档/2026-05-四科与前任管理员/](99-归档/2026-05-四科与前任管理员/README.md)（前任管理员 TIM/SYS-PREV 增补、四科科长厚稿与策划矩阵；设定真值已转正）。

其他历史剧本若仍位于 `.草稿-编写正文时不许修改/`，迁入时要按版本号分子文件夹，并补充 `canon_refs` 字段，标明所依赖的设定 `id`。

## 命名约定（生产稿专用）

生产稿**不必**沿用设定真值的 `{id}-{中文简名}.md`（见 [治理/01-命名文件与ID规则.md](../治理/01-命名文件与ID规则.md)），以免与「剧情实例」混淆；建议在文件名中体现**版本或批次**与**内容类型**，例如：

- **简报**：`v0.2-M01-仓库区谈判-简报.md`、`2026Q2-活动名-简报.md`  
- **脚本**：`M01-S03-对峙-脚本.md`、`支线-某NPC-台词草案.md`  
- **节拍**：`第一幕-节拍表.md`、`任务链-M01-M05-大纲.md`  

**可选 frontmatter**（便于导出与溯源）：

```yaml
canon_refs: [SYS-B9002, ORG-HRDEPT]
status: draft | frozen | deprecated
spoiler_tier: early | mid | late   # 见 治理/02-可见性与剧透分级.md
```

归档迁移到 `99-归档/` 时，建议再套一层版本目录，例如 `99-归档/v0.1-冻结批次/`，保留原文件名或附 `supersedes` 说明。
