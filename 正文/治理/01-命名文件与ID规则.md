# 命名、文件与标识符规则

## 第一条　标识符格式

设定条目的稳定标识符格式为：`前缀-主体`，全大写 ASCII，示例如下。

| 前缀 | 含义 | 示例 |
|------|------|------|
| WLD | 世界基石 | [WLD-00](../设定真值/00-基石/WLD-00-世界纪年与治理总纲.md) |
| ORG | 组织与势力 | [ORG-HELIOS](../设定真值/10-百科/组织/赫利俄斯/ORG-HELIOS-赫利俄斯集团.md) |
| PLC | 地点与空间类型 | [PLC-DISTRICT](../设定真值/10-百科/地点/PLC-DISTRICT-街区分层与层级-总述.md) |
| SYS | 系统与机制 | [SYS-B9002](../设定真值/10-百科/系统/SYS-B9002-桥接体与调度核心职能.md) |
| ENT | 文化与概念实体 | [ENT-CULTURE](../设定真值/10-百科/文化/ENT-CULTURE-企业文化与话语口径.md) |
| NAR | 叙事合同 | [NAR-00](../设定真值/20-叙事合同/NAR-00-叙事母题与基调.md) |
| PCM | 主角与主线（主角视角的设计母本，非逐字剧情） | [PCM-00](../设定真值/30-主角与主线/PCM-00-使用范围与边界.md) |
| TIM | 时序与历史 | [TIM-00](../设定真值/40-时序与历史/TIM-00-时序索引.md) |
| CHR | 人物（制度锚点） | [CHR-SOLEN-ELI](../设定真值/10-百科/人物/CHR-SOLEN-ELI-伊莱·索伦.md) |

## 第二条　文件名

Markdown 文件名建议为：`{id}-{中文简名}.md`，便于人类浏览与工具检索并存。

## 第三条　Frontmatter 必填字段

```yaml
id: ORG-EXAMPLE
title: 示例组织
status: active | draft | deprecated
visibility: internal | partner | public
```

扩展字段允许：`aliases`、`supersedes`、`related`。

## 第四条　中文标题

`title` 要为正式中文称谓，与对外宣传用语保持可追溯关系；若对外使用英文名，在 `aliases` 中注明。

## 第五条　内部链接

正文互引须符合 [03-内部链接书写规则.md](03-内部链接书写规则.md)；合并前运行 [管线/检查/check-links.py](../管线/检查/check-links.py)。

## 第六条　CHR 条文节序（建议）

人物条文（`CHR-*`）建议采用下列节标题，与 [TIM-04](../设定真值/40-时序与历史/TIM-04-前管理者执政与第九区事态背景年表.md) 等时序条文分工：**时间线**写人物经历锚点，**不**用「年表锚点」；**职权与牵制**写职务权柄与科层博弈，**不**用「制度接口」。

| 节 | 标题 |
|----|------|
| 第一条 | 登记与职务 |
| 第二条 | 玩家情报分层（B-9002） |
| 第三条 | 时间线 |
| 第四条 | 职权与牵制 |
| 第五条 | 关联条文 |
