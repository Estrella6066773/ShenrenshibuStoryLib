# 内部链接书写规则

本规则约束 `正文/` 下 Markdown **相对路径互引**，与 [01-命名文件与ID规则.md](01-命名文件与ID规则.md)（稳定 `id`）、[id-registry.yaml](id-registry.yaml)（登记路径）配合使用。

## 第一条　格式（硬约束）

1. 标准写法：`[显示文字](相对路径/文件名.md)`；显示文字优先用稳定 **`id`** 或正式标题简称。  
2. **禁止**把 `../` 写在方括号里：方括号内只能放显示文字，相对路径必须写在紧随其后的圆括号里（`]` 与 `(` 不可省略）。  
3. 路径使用 **正斜杠 `/`**；不要写 Windows 反斜杠。  
4. 锚点可选：`(.../文件.md#某标题)`；校验时只检查 `#` 前的文件是否存在。  
5. 外链（`http://`、`https://`、`mailto:`）与文内锚点（`#` 开头）不在本规则校验范围内。

## 第二条　路径怎么写（推荐）

| 场景 | 做法 |
|------|------|
| 同目录互引 | 仅文件名，如 `ORG-HRDEPT-人类事务管理部.md` |
| 跨子目录 | 从**当前文件所在目录**数层级，写 `../`；不确定时用下文「层级速查」 |
| 从任意设定真值条文指回治理 | 按实际深度写，例如系统条文 → `../../../治理/02-可见性与剧透分级.md` |
| 从 `正文/` 根导航指入子树 | 以 `正文/` 为基准，例如 `设定真值/00-基石/WLD-00-….md`（见 [00-START-HERE.md](../00-START-HERE.md)） |
| 登记过的正式条目 | 路径须与 [id-registry.yaml](id-registry.yaml) 中 `path` 字段一致（相对 `正文/`） |

**不要**凭记忆抄写别处的 `../` 层数；移动或新建子目录后，必须按**新位置**重算。

## 第三条　层级速查（设定真值 · 10-百科）

自 `正文/设定真值/10-百科/` 起算，到达下列目录需向上的 `../` 个数：

| 源文件所在目录 | → `00-基石/` | → `40-时序与历史/` | → `治理/` |
|----------------|-------------|-------------------|-----------|
| `系统/`、`文化/`、`地点/`、`人物/` | `../../` | `../../` | `../../../` |
| `组织/`（如反抗军、运动） | `../../` | `../../` | `../../../` |
| `组织/帮派/` | `../../../` | `../../../` | `../../../../` |
| `组织/赫利俄斯/` | `../../../` | `../../../` | `../../../../` |

同层互引（如 `系统/` → `地点/`）用 `../地点/PLC-….md`。

## 第四条　改稿与登记流程

1. **新建或移动** Markdown 后：更新 [id-registry.yaml](id-registry.yaml) 的 `path`（若为该库正式条目）。  
2. 用编辑器或脚本检查从该文件发出的链接；合并前运行 [管线/检查/check-links.py](../管线/检查/check-links.py)。  
3. 批量改名、目录重组（例如组织条文迁入 `组织/赫利俄斯/`）时，须**全库检索**旧路径片段（如 `组织/ORG-HELIOS-`），不可只改索引页。  
4. 叙事生产稿引用设定真值时，优先写**指向真值文件**的相对路径，并在 frontmatter 用 `canon_refs` 列出 `id`（见 [叙事生产稿/README.md](../叙事生产稿/README.md)）。

## 第五条　自动校验与提交前自动修正

- 脚本：[管线/检查/check-links.py](../管线/检查/check-links.py)（含注册表路径与 `manifest.yaml` 的 `canon_ids` 核对）  
- 建议时机：提交前、升版 [玩家投放/](../玩家投放/README.md) 前、目录重组后。  
- **Git 提交前**（已配置 `core.hooksPath` 时）：自动执行 `--fix --stage-fixed`——按链接文字中的 **`id`**、**文件名**或**全库唯一 basename** 查找目标并重写相对路径，将 `正文/` 变更加入暂存区；**不会**自动 `push`。  
- 手动：`python 正文/管线/检查/check-links.py --fix`（只改文件）或加 `--stage-fixed`（等同钩子）。  
- 清单入口：[管线/00-导出与一致性检查.md](../管线/00-导出与一致性检查.md) 第二条。  

无法自动修复时仍会拒绝提交（同名多文件、目标不存在、注册表／manifest 问题等）。修正后建议看一眼 diff。

## 第六条　常见错误对照

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 方括号内带 `../`（缺 `](`） | 渲染器不识别为链接 | 先 `]` 再 `(`，路径写在括号内 |
| 从 `组织/帮派/` 用 `../../00-基石/` | 指向 `10-百科/00-基石/`（不存在） | `../../../00-基石/` |
| 从 `组织/赫利俄斯/` 用 `../../00-基石/` | 同上 | `../../../00-基石/` |
| 文件名与仓库不一致（如 `05-四科冲突…`） | 断链 | 以磁盘文件名为准 |
| 指向已删除的旧目录 | 断链 | 查 registry 或 [10-百科/README.md](../设定真值/10-百科/README.md) 索引表 |
