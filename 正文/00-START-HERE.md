# 阅读起点（本库导航）

本库面向**长期协作**与**大规模玩家考据**两类读者。请先理解分层，再进入具体条文。

## 一、五层结构（职责不可混用）

| 层级 | 路径 | 用途 |
|------|------|------|
| 设定真值 | [设定真值/](设定真值/README.md) | 内部完整世界观与机制；写作与审核的**唯一锚点**。 |
| 叙事合同 | [设定真值/20-叙事合同/](设定真值/20-叙事合同/README.md) | 母题、基调、红线、终局分型；约束一切对外叙事口径。 |
| 叙事生产 | [叙事生产稿/](叙事生产稿/README.md) | 任务简报、脚本、关卡叙事等**可过期**生产稿。 |
| 对玩家冻结 | [玩家投放/](玩家投放/README.md) | 已批准对外披露内容，**按版本目录**归档。 |
| 治理与管线 | [治理/](治理/00-协作与改稿原则.md)、[管线/](管线/00-导出与一致性检查.md) | 命名、ID、可见性、导出与一致性检查。 |

## 三、考据与检索约定

- 每一条正式设定均配有稳定 **`id`**（见各文 frontmatter 与 [治理/id-registry.yaml](治理/id-registry.yaml)）。  
- 同一事实不得在多处互斥陈述；若生产稿与真值冲突，以 [设定真值/](设定真值/README.md) 为准，除非该生产稿已标明为**平行草案**并获特批。

## 四、改稿权限

凡修改 [设定真值/](设定真值/README.md) 或 [玩家投放/](玩家投放/README.md)，须遵守 [治理/00-协作与改稿原则.md](治理/00-协作与改稿原则.md) 所列流程与可见性约定。
