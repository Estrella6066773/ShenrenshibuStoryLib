# 企业辖区的空间分层模型

## 第一条　模型之目的

本条文界定单一企业辖区内城市空间之**分层叙事模型**，用于统一任务发生地、监控密度、法治外观与灰色经济之逻辑，避免「同一街区在不同文本中自相矛盾」。

## 第二条　三层结构

### A 层：明亮层（企业橱窗区）

监控与执法无人机之部署密度高，消费与医疗资源在宣传口径上充足，对外构成**秩序与繁荣之橱窗**。叙事上可用于：体面交易背后之肮脏、舆论焦点、企业形象工程。

### B 层：缝隙层（居住与灰业混合区）

普通住户、小商贩与外包劳工混居，规则弹性大，冲突可通过谈判、贿赂或暴力等多路径解决。为**委托任务、社区纠纷与势力渗透**发生之主要场域。

### C 层：暗面层（维护通道、废弃区与黑市网络）

基础设施老化，系统覆盖存在断层，非法实验、黑市集散与隐藏事故遗留地集中。叙事上可用于：帮派据点、反抗组织藏匿、被企业刻意淡化之事件现场。

## 第三条　关键公共空间（制度—叙事接口）

下列地点为辖区内反复出现之**公共空间类型**；具体地理命名可由项目再行赋值，但功能类型须与本条一致：

| 类型 | 叙事功能概要 |
|------|----------------|
| 合同集中办理场所 | 劳务、债务、缓刑劳动等契约之集中签署；与「人类签字背锅」机制强相关。见 [PLC-CONTRACTHALL](../10-百科/地点/PLC-CONTRACTHALL-合同大厅.md)。 |
| 维修井口与通风塔 | 连接 C 层之物理接口；追踪、伏击、藏匿之典型场景。见 [PLC-MAINTENANCE](../10-百科/地点/PLC-MAINTENANCE-维修井口与暗面通道.md)。 |
| 数据当铺 | 以隐私、记忆片段或账号权益换取流动性；黑客、舆情与勒索线之枢纽。见 [PLC-DATAPAWN](../10-百科/地点/PLC-DATAPAWN-数据当铺.md)。 |
| 无人诊所 | 自动化医疗节点；伪造病历、药品走私与人体改造等灰线之接口。见 [PLC-CLINIC](../10-百科/地点/PLC-CLINIC-无人诊所.md)。 |
| 看板广场 | 巨型屏幕投放企业口号与关键绩效指标；舆情事件与群体骚乱之放大器。见 [PLC-PLAZA](../10-百科/地点/PLC-PLAZA-看板广场.md)。 |
