# 辖区空间与城市分片（第九区·工业拆分·郊区不接商业）

## 第一条　适用范围与功能

本条文约定**城市 A（`[城市A]`）** 与 **第九区** 之间的空间关系，以及第九区内部由**商业区、居民区、幸存工业区、郊区**四片与已定名 **废墟区**（由原工业区主体大部固化而来）构成的**统一建成叙事**，用于统一书写任务发生地、管控强度、补给走廊与灰色经济，避免同一街区在不同稿件中互相矛盾。

**废止说明**：旧版「明亮层／缝隙层／暗面层」「高／中／低合规带」及 **A/B/C 叙事分层** 不再作为真值；若旧稿残留上述用语，应以本条为准改写。凡旧稿写「工业区整块隔主河对岸」而与本条**工业已拆分、主体转废墟**冲突者，以本条为准改写。

## 第二条　城市 A：第一区与第九区

- **第一区**：城市 A 的**核心辖区**，承担总部、金融—服务业主心与高合规城市形象；叙事地形**可参照**芝加哥都会核心（湖滨高密度、枢纽与总部带），**具体洋名与街区名由项目另定**。  
- **第九区**：位于**第一区外缘**、**面向自由人联合阵线武装压力方向的桥头堡分辖区**；叙事地形**可参照**芝加哥都会西南外缘走廊节点（如乔利埃特式的河—铁—州际主轴），**具体地名由项目另定**。两区之间以制度上的辖区边界与检查站叙事衔接，本条不钉死公里数。

## 第三条　水系与主轴

第九区按一条**进城主轴走廊**贯穿（高速与货运铁路叠合或并行，自第一区方向刺入，再伸向阵线前沿）；**主干河流或运河**与主轴**横交**，与**铁路路堑**共同切成可画的块片。

真值层锁定**主干河道**在平面图上的**总体走向**为**自西南向东北**蜿蜒；**水流**沿程指向**东北**，即**流向第一区方向**。在接近第一区的一端（叙事制图常取**北端**），河道轴线相对**正北**再**向东偏转约 20°**——可读作末端折向**东北偏东**，与城门／检查站叙事衔接时更易形成「进城弯道」。支流、局部裁弯取直仍可由**叙事生产稿**细化。

另有一条**较窄河道**，专门隔开 **幸存工业区** 与 **废墟区（原工业区主体）**——尺度上明显窄于主干河道，便于叙事上写成「一眼能望对岸厂区／废墟轮廓」的间隔。

## 第四条　第九区建成地带：一体化拼图（制图默认上北下南）

叙事把全区读作**同一张地图**，关键约束如下：

1. **商业区—居民区**：沿主轴入门一侧连片：**商业区**邻第一区方向，向内紧贴 **居民区**；人事部历代本部仍在 **居民区**（见 [ORG-HRDEPT](../10-百科/组织/赫利俄斯/ORG-HRDEPT-人类事务管理部.md)）。  
2. **郊区（贫民区）**：辖区官方／地图称 **郊区**，实质仍为贫民区；**不得与商业区直接接壤**（二者之间必有 **居民区** 和／或 **主干河道** 等隔开）。**郊区须与废墟区直接接壤**，以便外缘缓冲、走私与废墟叙事叠合。  
3. **工业区（幸存）**：仅余**北方小块**，仍为围墙化厂区、联运与仓储；与南侧 **原工业区主区域**（现已大部为 **废墟区**）之间隔 **较窄河道**。  
4. **废墟区**：以**原工业区主区域大部**为主体，事变遗留固化；可与 **居民区西侧** 形成连续的滨河断裂带（人事部旧中枢等叙事见 [PLC-RUINS](../10-百科/地点/PLC-RUINS-废墟区-第九区事变遗留片区.md)）。**主干河道**仍可在叙事上裁切出**跨河碎片**，与铁路、路堑叠加形成地面断点；**河底或沿岸地下隧道**（遗留或新挖）用于勾连碎片与 **幸存工业区后门—郊区—居民区西侧** 的走私／追捕链。

**勿与「第一区」混淆**：「第一区」是城市 A **核心分片**；本条称谓仅描述**第九区内部**。

| 真值称谓 | 口径与意象（写作锚） | 合规与管控倾向 |
|----------|---------------------|----------------|
| **商业区** | 商业街面、写字楼、会展中心式门面、沿河橱窗岸线 | **最高**：监控与身份核验偏密 |
| **居民区** | 住宅、学校医院、小店与底商、一般街道网格 | **居中**：委托与社区纠纷主战场 |
| **幸存工业区** | 北方小块仍在运转的厂区、联运接口 | **中高**：货运与安全叙事偏重 |
| **废墟区** | 原工业主体大部＋事变遗留；非单一色块可含跨主干河的碎片 | **视情节**：禁区／灰带 |
| **郊区** | **官方／地图定名**；棚户、外缘缓冲，**实质为贫民区** | **最低**：外包武装与损耗叙事 |

### 商业区（邻第一区）

小街块旧核与河岸橱窗段、合规中枢与对外窗口集中；暴力处置需**合法化包装**。**不与郊区直接相邻**；门面叙事面向主轴与河道，而非贴贫民外缘。

### 居民区（主体）

住宅与一般街道交织；**西侧**常与 **废墟区** 滨河带接壤或隔窄岸相望。铁路路堑下自然形成**低可见度通道**；**不再使用「暗面层」旧称**。

### 幸存工业区（北方小块）

位于**原工业区地带的北端残余**；与南侧 **废墟区（原工业主体）** 以 **较窄河道** 分隔，仍保留装卸、铁路支线与企业安保叙事。**具体厂名由项目另定**。

### 郊区（官方定名；实质仍为贫民区）

与 **废墟区** 共有边界，承担外缘棚户、临时装卸与阵线张力叙事；**不与商业区**划界相邻。写对白时：**职员口径「郊区」**、**街头与反抗话术「贫民区」**。

## 第五条　公共空间类型（制度—叙事接口）

下表列出辖区内会反复出现的**公共空间类型**；具体地名可由项目另行命名，但功能类型要与本表一致。

| 类型 | 叙事功能概要 |
|------|----------------|
| 契约集中签署节点 | 劳务、债务、缓刑劳动等契约集中签署；与「以人类签字承接可归责后果」的机制密切相关。 |
| 路堑与地下连廊节点 | 路堑、河底管廊、封闭匝道下、井口与通风塔等**物理上低可见**的连通点；便于追踪、伏击与藏匿（**不再**使用「暗面连通维修节点」旧称）。 |
| 数据质押与隐私贴现节点 | 以隐私、记忆片段或账号权益换取流动性；黑客、舆情与勒索线的枢纽。 |
| 自动化医疗灰接口 | 自动化医疗节点上的灰色接口；伪造病历、药品走私与人体改造等叙事可由此接入。 |
| 舆情宣传放大节点 | 巨型屏幕滚动企业口号与 KPI；可作为舆情事件与群体骚乱的放大器。 |

除了已定名的 **废墟区** 以外，上表名称都是**叙事功能类型**，并不是已敲定的地标名称；其余地名可在后续版本中赋值。
