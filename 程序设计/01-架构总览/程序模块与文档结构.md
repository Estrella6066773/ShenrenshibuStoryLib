## 一、代码模块（当前）

| 模块键 | 路径 `Assets/01_Scripts/` | 职责 |
|--------|---------------------------|------|
| CoreMessaging | `Core/Messaging/` | 全局消息总线、启动门控 |
| PlayerEconomy | `PlayerState/` | 资金、日历时钟、日结 |
| Employee | `Employee/` | 员工数据、状态、标签 Buff |
| Assignment | `Assignment/` | 委托 Tick、分支、任务交互效果 |
| Conflict | `ConflictSystem/` | 矛盾累积、模板、邮件桥接 |
| Stress | `StressSystem/` | 压力分段、事件、邮件桥接 |
| MessageSystem | `MessageSystem/` | 邮件 / 聊天 / 日志 UI 与 SO |
| DialogSystem | `DialogSystem/` | 对话节点播放（部分类型在 `DialogSystem` 命名空间） |
| Resume | `Resume/` | 简历窗口、拖拽、排序 |
| Portrait | `Portrait/` | 证件照生成与分层 |
| IdleArea | `IdleArea/` | 待命区收纳 |
| UI_Navigation | `UI/Navigation/` | 底部导航壳 |
| EditorTools | `Editor/` | 菜单、Inspector、场景工具 |
| RuntimeTests | `Tests/Runtime/` | Play 模式验证辅助 |
| DataAssets | （数据在 `04_Data/`） | SO、对话与邮件库 |

模块依赖摘要见 `Docs/ShenrenshibuStoryLib/系统设计/03-计划/计划-模块依赖与建议顺序.md`。

### 与机魂 Assembly 的对照（未来可选）

| 机魂 | 神人事部（现状） | 建议演进 |
|------|------------------|----------|
| `Game.Core` | `Core/` + 各模块共享类型 | 抽离无 UI 的 Core 程序集 |
| `Game.Simulation` | 无独立仿真核 | 委托 Tick 留在 Assignment |
| `Game.Presentation` | 各模块 `View/` | 可选 `Presentation` 程序集 |

## 二、文档：为何保留「实现-*」按模块命名

顶栏按**模块**分类，便于「改行为 → 改对应 `实现-*`」：

| 文档类型 | 路径模式 |
|----------|----------|
| 需求 | `Docs/ShenrenshibuStoryLib/系统设计/02-需求/需求-<模块>.md` |
| 实现 | `Docs/ShenrenshibuStoryLib/程序设计/04-实现/实现-<模块>.md` |
| 追溯 | `Docs/ShenrenshibuStoryLib/程序设计/05-交付/交付-追溯矩阵.md` |

**不建议**仅用「UI / 逻辑 / 数据」取代模块名；可在 `Docs/ShenrenshibuStoryLib/程序设计/02-运行时逻辑/` 增加第二层索引（待建）。
