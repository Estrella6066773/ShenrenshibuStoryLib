# Unity 工程集成说明

《神人事部》Unity 工程是本文档库的**实现载体**，不是第二套文档根。

## 角色划分

| 载体 | 职责 |
|------|------|
| **ShenrenshibuStoryLib（本仓库）** | 系统设计、程序设计、叙事正文、治理与 CI |
| **神人事部（Unity）** | `Assets/` 代码与数据、`ProjectSettings/`、UVC 协作 |

## 克隆与初始化

```bash
git clone <神人事部-Unity-仓库>
cd 神人事部
git submodule update --init --recursive
```

文档工作目录：`Docs/ShenrenshibuStoryLib/`（本仓库检出路径）。

## 版本控制分工（与机魂一致）

- **Git**：`Docs/ShenrenshibuStoryLib/`、`.github/`、`Scripts/Get-DocsDate.ps1` 等  
- **UVC**：`Assets/`、`ProjectSettings/`  

`ignore.conf` 已排除 `/.git`、`/.github`，避免双端冲突。

## 改文档后的习惯

1. 在本库对应分支提交并推送（设定正文走 `正文/治理/` 流程）。  
2. Unity 主仓库更新子模块指针（`git add Docs/ShenrenshibuStoryLib`）。  
3. 若改行为：同步 `系统设计/02-需求` 与 `程序设计/04-实现`，并更新 `05-交付/交付-追溯矩阵.md`。

## 历史

- 叙事内容原在独立 StoryLib；2026-05 起将 Unity 工程文档并入本库，形成统一文档库。
